Direct API integration

Point any OpenAI-compatible client or editor extension at the endpoint above. First mint a key on the API Keys page (it's shown once), then drop it into the config below in place of <YOUR_KEY>.

VS Code — Continue extension

Install Continue, then add this model to ~/.continue/config.yaml: 

models:
  - name: seko ({{ llm_model }})
    provider: openai
    model: {{ llm_model }}
    apiBase: {{ llm_base_url }}
    apiKey: <YOUR_KEY>
    roles:
      - chat
      - edit

VS Code — Cline (agentic)

Prefer agentic, multi-file editing? In Cline's settings choose API Provider OpenAI Compatible, set Base URL to {{ llm_base_url }}, API Key to your seko key, and Model ID to {{ llm_model }}. Cline leans on the model's tool-calling, so expect heavier token use than plain chat.

Any language — curl

curl {{ llm_base_url }}/chat/completions \
  -H "Authorization: Bearer $SEKO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "{{ llm_model }}", "messages": [{"role": "user", "content": "Hello"}]}'

Any language — OpenAI SDK (Python)

from openai import OpenAI

client = OpenAI(base_url="{{ llm_base_url }}", api_key="<YOUR_KEY>")
resp = client.chat.completions.create(
    model="{{ llm_model }}",
    messages=[{"role": "user", "content": "Hello"}],
)
print(resp.choices[0].message.content)

Codebase indexing (Zoo Code)

The homelab also serves a local embeddings model plus a shared Qdrant vector database, so editor extensions can do semantic “find the code that does X” retrieval over your whole repo. Your seko key already works for embeddings — no separate key needed.

Qdrant is shared across all users — collections are named per project but are mutually visible, so don’t index anything you wouldn’t share.

VS Code — Zoo Code (Roo/Kilo-style indexing)

In Zoo Code open Settings → Codebase Indexing and set:

  • Embedder provider: OpenAI Compatible
  • Base URL: {{ llm_base_url }}
  • API key: your seko key
  • Model: {{ embedding_model }}
  • Embedding dimension: {{ embedding_dimension }}
  • Qdrant URL: {{ qdrant_url }}{% if qdrant_api_key %} · Qdrant API key: as above{% endif %}

Then click Start indexing.

Reaching Qdrant needs LAN/Tailscale access to {{ qdrant_url }}. If your Tailscale access is limited to your workspace SSH port, run the editor inside your hosted workspace, which can reach it directly.

Verify — curl

curl {{ llm_base_url }}/embeddings \
  -H "Authorization: ******" \
  -H "Content-Type: application/json" \
  -d '{"model": "{{ embedding_model }}", "input": "hello"}'

A {{ embedding_dimension }}-length vector in data[0].embedding means you’re set.
